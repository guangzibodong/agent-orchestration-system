# Feature Brief Template

Use this template before any new engineering slice.

## 1. Title

Short user-facing feature name.

## 2. Target User

Who is this for?

## 3. Problem

What pain does the user have today? Include the current workaround.

## 4. Value Hypothesis

If we solve this, what behavior should improve?

## 5. Scope

What is included in this slice?

## 6. Non-Goals

What are we explicitly not doing?

## 7. User Flow

Step-by-step success path.

## 8. Failure And Edge Cases

Include unsafe repository, missing agent configuration, timeout, cancelled job, failed gate, retry, and persistence when relevant.

## 9. UI / API Behavior

Screens, components, API endpoints, payloads, and state changes.

## 10. Acceptance Criteria

Use verifiable statements:

- Given ...
- When ...
- Then ...

## 11. Test Plan

Commands, unit/integration coverage, smoke checks, UI checks.

## 12. Data And Operations Impact

Schema changes, artifacts, cleanup, env vars, deployment, rollback.

- [ ] Needs product clarification
- [ ] Needs UI clarification
- [ ] Deferred
